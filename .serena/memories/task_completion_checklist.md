# 任务完成检查清单

当完成代码修改任务后，应该执行以下步骤：

## 1. 代码格式化
```bash
go fmt ./...
```

## 2. 代码质量检查
```bash
# Lint 检查
golint ./...

# Vet 检查（检测可疑的代码构造）
go vet ./...
```

## 3. 构建验证
```bash
# Linux
make

# Windows
build.bat
```

## 4. 测试运行
- 虽然项目中没有单元测试，但应该手动测试修改的功能
- 确保 billing 服务可以正常启动
- 检查日志文件 `billing.log` 是否有错误

## 5. 提交前检查
- 确保没有引入编译错误
- 确保遵循项目的代码风格
- 确保中文注释清晰明了
- 确保没有提交敏感信息（如密码、密钥等）
